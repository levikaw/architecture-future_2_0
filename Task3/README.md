# Задание 3. Разработка технологического радара и роадмапа изменений для поддержки бизнес-целей компании



## Технологический радар

### Текущие технологии и методологии

| Технология/Методология|  Описание|
|---|---|
| Microsoft SQL Server| DWH для хранения данных|
| Power BI| BI-инструмент для визуализации данных. Подготовка отчетов, аналитика по данным клиентов|
| Apache Camel| Интеграционная шина для связи между системами|

### Планируемые технологии и методологии

| Технология/Методология| Описание|
|---|---|
| Облачное хранилище (e.g., AWS)  | Хранение данных в облаке для оптимизации ресурсов. Поддерживает простое масштабирование и способствует увеличению скорости подготовки отчетности|
| Микросервисная архитектура| Разделение на независимые сервисы способствует независимому развитию доменов|
| Apache Kafka| EDA для взаимодействия между различными микросервисами|
| Data Mesh| Децентрализованный подход к управлению данными, способствует облегченной интеграции новых бизнес-проектов|

## Роадмап

| Этап| Ожидаемые результаты| Ответственные команды| Обоснование изменения|
| ---| ---|---| ---| ---|
| Аудит текущей инфраструктуры   | Понимание сильных и слабых сторон | Архитекторы, ИТ команда  | требуется для формирования ясной картины существующих процессов и технологий, что позволит определить направления для улучшения|
| Миграция в облако | Увеличение масштабируемости и гибкости инфраструктуры| ИТ команда| Снижение затрат и улучшение доступа к данным для повышения гибкости и возможности быстрого масштабирования|
| Интеграция с внешними партнерами    | Обогащение сервисов и создание дополнительных возможностей | Архитекторы,Команда интеграций| Освоение новых бизнес-направлений |
| Разработка моделей управления данными | Улучшение оперативности и качества работы с данными    | Архитекторы, ИТ команда| формирование гибкой системы управления данными позволит быстро адаптироваться к изменяющимся требованиям и обеспечит поддержку всех бизнес-направлений |
